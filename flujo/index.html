Flujo de Autenticacion
3 metodos de login: Local (email/usuario + password), Google OAuth, y Recuperacion de contraseña.
🔑
Login Local
POST /api/auth/login
Usuario ingresa email/usuario + password
↓
validateLogin middleware (sanitizacion)
↓
checkBruteForce (5 intentos/15min → bloqueo 30min)
↓
Busqueda: email directo O username (regex exacto)
↓
Verificar isActive (baneado → 403)
↓
bcrypt.compare() → logEvent(LOGIN)
↓
Retorna JWT (24h) + userData → localStorage
🌐
Google OAuth
POST /api/auth/google
FirebaseInitService carga config desde /api/config
↓
signInWithPopup(FirebaseAuth, GoogleProvider)
↓
Obtiene idToken de Firebase
↓
POST /api/auth/google { idToken }
↓
Verificacion: Identity Toolkit API → JWKS fallback
↓
Find or Create user (password random)
↓
Retorna JWT + userData
Fallback: si popup bloqueado → signInWithRedirect
📝
Registro
POST /api/auth/register
validateRegister (name min2, email regex, pass min8, no spaces)
↓
Sanitizacion: strip <>, '";\, $
↓
Verificar email unique
↓
UserModel.create() → bcrypt auto-hash
↓
logEvent(REGISTER) + JWT
🔄
Recuperar Contraseña
2 pasos
Paso 1: POST /forgot-password { email }
↓
Genera codigo de 6 digitos → resetToken (15min expiry)
↓
Retorna codigo en response (sin email service)
↓
Paso 2: POST /reset-password { email, code, newPassword }
↓
Verifica codigo + no expirado → update password
↓
Retorna nuevo JWT
🎫
Ciclo de Vida del JWT
Token-based authentication
Emision:
  • Register, Login, Google Login
  • Reset Password, Heartbeat (chat)
Payload:
  • { id: userId }
  • Expiry: 86400s (24 horas)
Refresco:
  • Chat heartbeat cada 30s
  • Devuelve nuevo JWT en response
Validacion:
  • AuthInterceptor: agrega Bearer header
  • AuthGuard: verifica exp en payload

Capas de Seguridad
Proteccion multi-capa: Helmet, Rate Limiting, Brute Force, Input Sanitization, Attack Pattern Blocking.
🛡️
Helmet + Headers
  • Content Security Policy (CSP) — permite tailwind, FontAwesome, Google Fonts
  • Strict-Transport-Security (HSTS)
  • X-Content-Type-Options: nosniff
  • X-Frame-Options: DENY
  • X-XSS-Protection: 1; mode=block
  • Referrer-Policy: strict-origin
  • Permissions-Policy restricciones
⏱️
Rate Limiting
  • Global: 100 requests / 15 min por IP
  • Auth endpoints: 10 requests / 15 min por IP
  • Admin endpoints: 60 requests / 60s por IP
  • IPv6 compatible (ipKeyGenerator)
🔒
Brute Force Protection
Intento de login fallido
↓
Incrementar contador IP en memoria
↓
5 intentos / 15 min → Bloquear IP
↓
Bloqueo dura 30 minutos
↓
Cleanup automatico cada 10 min
🧹
Input Sanitization
  • express-mongo-sanitize: reemplaza $ y . por _
  • hpp: HTTP parameter pollution protection
  • Custom: strip <>, '";\, $ de inputs
  • Validation: email regex, password min/max, name length
🚫
Attack Pattern Blocking
Bloquea request que coincidan con patrones de ataque
Path Traversal:
  • \.\./ \.\.\%2f %2e%2e%2f
  • \.\.\\ \.\.\%5c
SQL Injection:
  • union.*select, select.*from
  • insert.*into, delete.*from
  • drop.*table, update.*set
XSS:
  • <script, javascript:
  • onload=, onerror=, onclick=
Command Injection:
  • cmd=, exec=, system(
  • passthru, shell_exec
Sistema Real-Time (Polling)
No usa WebSockets. Todo funciona con HTTP polling a intervalos variables.
💬
Chat Polling
3 intervalos simultaneos
cada 30s → POST /chat/heartbeat
Refresca JWT + marca usuario online
+ 30s
cada 5s → GET /chat/conversations
Recarga lista (unread counts, last message)
+ 5s
cada 2s → GET /chat/conversations/:id/messages
Detecta nuevos mensajes por diff de IDs
📋
Logs Polling
Superadmin only
cada 5s → GET /admin/logs?since=timestamp
Carga nuevos logs desde ultima consulta
↓
Filtra duplicados por _id
↓
Prepende a la lista (max 100)
↓
Actualiza lastTimestamp
🟢
Online Status
In-memory tracking
  • Almacenado en Map<userId, { lastSeen }> en memoria del servidor
  • Se actualiza en: heartbeat, conversation list fetch
  • Expuesto via GET /chat/online
  • Cada conversation incluye memberDetails[].isOnline
  • No persiste en DB — se pierde al reiniciar servidor
📡
Seed Data en Startup
Datos iniciales automaticos
  • ensureSuperAdmin(): si no hay superadmin, promueve al primer admin
  • seedProducts(): si DB vacia, crea 5 productos (Panel VIP, Bypass, Proxy...)
  • patchProductImages(): asegura que Bypass APK tenga imagen
  • seedChatData(): crea 4 canales (general, soporte, anuncios, off-topic) + 6 notificaciones
